Build or rebuild the target repository primer and verification profile.

1. Read `AGENTS.md` and treat its declared workspace as the target. The embedded projection expands this instruction to the target and harness operating cards.
2. Use the harness-relative output paths `docs/target-repo-primer.md` and `runtime/repository-checks.json`. In an embedded installation, Cursor filesystem operations use `.pancreator/docs/target-repo-primer.md` and `.pancreator/runtime/repository-checks.json`, while CLI arguments retain the harness-relative paths.
3. In an embedded installation, run `./.pancreator/bin/pan technologies detect --json` before delegation. Declare the detected language handbooks under `.pancreator/governance/handbooks/target/<language>/style-guide.md`, `.pancreator/governance/policies/LANG-001.json`, and the marked LANG-001 lookup rows as librarian outputs.
4. Invoke the `librarian` subagent with the target workspace root, both output paths, detector output, and declared generated-governance outputs. Require it to apply `PRIMER-001` and `REPO-001`, create the primer when absent or regenerate it when present, and read existing outputs first when they exist.
5. Require the librarian to inventory target-owned documentation, inspect every document likely to contain primer-relevant administrative, architectural, interface, structure, or gotcha information, and incorporate useful verified details into the appropriate primer sections. It must also inspect representative code, setup/build/install/test scripts, manifests, and bounded `git log` history when available, reconciling stale documentation against executable sources and current code.
6. Require the librarian to populate repository-check profiles only from verified target conventions. `fast` must use the shortest documented default/primary suite and must not duplicate `full` when distinct commands exist; separately documented slow or integration checks belong in `secondary`; `full` must cover all documented suites. Commands that depend on an interpreter, virtual environment, SDK, compiler, or package manager must use the explicit repository-declared entrypoint when available, should include identity/version probes, and should set `timeout_ms` when the repository states an expected duration. Empty profiles are valid; guessed commands are not.
7. Require the librarian to generate one marked target handbook per detected language, remove only stale marked language handbooks, and rewrite LANG-001 with the exact sorted handbook guidance sources. It must add idempotent LANG-001 lookup rows only for coder, reviewer, qa-tester, and spotfixer without changing unrelated rows.
8. Run `./bin/pan requirements run --persona librarian --workflow standalone --stage build-docs --kind documentation --registry TARGET-REPO-PRIMER-VALIDATE-001 --target docs/target-repo-primer.md --json`.
9. Run `./bin/pan repository-check validate --json`.
10. If either validation fails, provide the issues to the librarian for one correction attempt, then rerun both validations. Stop and surface unresolved issues if the second attempt fails.
11. Do not modify target source or workflow state and do not commit, push, merge, publish, or deploy.
12. Surface both validated paths, generated handbook ids, the source HEAD recorded by the primer/configuration, and a concise summary of the sections and profiles built or rebuilt.
