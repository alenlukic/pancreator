# Governance AGENTS Status v1 Intake Request

## Feature ID
`governance-agents-status-v1`

## Goal
Refresh AGENTS governance content to reflect current-state status accurately and ensure documented statements match repo reality.

## Scope (explicit files allowed)
- `AGENTS.md`

## Required outputs (spec/plan/touch-set + target docs/artifacts)
- **Spec:** concise status-refresh spec focused only on current-state accuracy.
- **Plan:** verification-first plan to compare AGENTS statements against repository reality.
- **Touch-set:** exact AGENTS sections that require status/current-state updates.
- **Artifacts:** updated `AGENTS.md` with status/current-state refresh only, aligned with present repo structure and phase reality.

## Acceptance criteria
- AGENTS status/current-state sections are factually aligned with the repository at edit time.
- Changes are limited to status/current-state refresh and consistency fixes.
- No new governance policy, workflow mandate, or delegation rule is introduced.
- Spec, plan, and touch-set are present and trace to the final edits.

## Out of scope
- Introducing new governance rules or changing existing policy intent.
- Editing files other than `AGENTS.md`.
