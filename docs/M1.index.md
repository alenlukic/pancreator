   active-memory refresh, commit/policy-compliance, and librarian pre-close validation.

   mode applies.

5. `docs/PRD.summary.md` and `docs/PRD.index.md` for product orientation and section

6. Full `docs/BOOTSTRAP.md` or `docs/PRD.md` only when the task requires phase-exit detail,
   product-spec depth, or line-anchored citation repair.


## Current bootstrap tracking state

`tesseract.yaml` is the live state tracker. It currently records
`bootstrap.phase: "5"`, `status: phase-5-in-progress`, and
`project_root: "."`. The `.` value means this harness is embedded in the same
repository it operates on.

Phase 5 is the right tracking phase because Phase 4 substrate, personas,
contracts, and the US-1 dogfood proof bundle are complete and ratified.
Pause/resume/abort evidence is captured, while Phoenix external trace
verification remains a deferred engineering item for `@tesseract/run-logger`
and `tesseract-engineer`. `tess run feature-delivery <inbox-entry>` (path
relative to `src/inbox/in/`, for example
`<day-bucket>/<SID>_<HHMM>_<slug>.md`‚Äînot the `src/inbox/in/` prefix) creates
the active-work `state.json`, `handoff.md`, `next-prompt.md`, and
`run.log.jsonl` scaffold for manual persona delegation. Stage progress is
recorded only through `tess advance` or explicit `tess repair-state`;
`refresh-prompt` can regenerate `handoff.md`/`next-prompt.md` from the current
ledger. Complete runs emit a bounded librarian handoff for agent-executed
artifact closure via `tess close-artifacts <task-id>`. Do not infer M1 completion from the phase number;
use `docs/BOOTSTRAP.md` Phase 5 exit criteria for ratification.

M1 is the Cursor-only, single-repo MVP. It lights up basic `feature-delivery`,
greenfield/adopt install paths, pause/resume/abort, package boundaries, one delivery
report artifact, and feature indexing. `chat-with-persona` runtime and standalone
`examples/` apps are backlog-deferred past M1.

this file plus `docs/PRD.summary.md` and `docs/PRD.index.md` answer the routing question.
