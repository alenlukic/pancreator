# M1 index — compact route into MVP and bootstrap scope

This file is the default route for M1 work. Read this before opening full
`docs/PRD.md` or `docs/BOOTSTRAP.md`.

## Default retrieval order

1. `AGENTS.md` for the live operating contract.
2. `OPERATION.md` for inbox lifecycle, feature-delivery loop, `pan` CLI verbs,
   active-memory refresh, commit/policy-compliance, and librarian pre-close validation.
3. `lib/memory/active/current.md` for current operator pointers, unless simple task
   mode applies.
4. This file for M1 scope and bootstrap routing.
5. `docs/PRD.summary.md` and `docs/PRD.index.md` for product orientation and section
   selection.
6. Full `docs/BOOTSTRAP.md` or `docs/PRD.md` only when the task requires phase-exit detail,
   product-spec depth, or line-anchored citation repair.

## M1 scope quick map

M1 is the Cursor-only, single-repo MVP. It lights up basic `feature-delivery`,
greenfield/adopt install paths, pause/resume/abort, package boundaries, one delivery
report artifact, and feature indexing. `chat-with-persona` runtime and standalone
`examples/` apps are backlog-deferred past M1.

| Need | Read first | Open full source only when |
|---|---|---|
| M1 package dependency order | This file §“M1 package order” | authoring or repairing phase contracts requires exact BOOTSTRAP wording |
| MVP product scope | `docs/PRD.summary.md`, then `docs/PRD.index.md` | the task changes scope or needs anchored PRD citations |
| Bootstrap phase sequencing | This file §“Bootstrap phase routes” | the task changes phase inputs, outputs, or exit criteria |
| Persona/subagent roster | `lib/personas/<name>.md` and `.cursor/agents/<name>.md` | authoring persona specs or fixing dual-anchor references |
| Context economy | `lib/memory/handbook/context-economy.md` | changing index policy or model/subagent policy |

## Token-economy rule

this file plus `docs/PRD.summary.md` and `docs/PRD.index.md` answer the routing question.

wording, changed scope, or dual-anchor citations.

compact `work/<day>/<task-id>/handoff.md` and delegate to the next persona
instead of carrying planner context into implementation.
