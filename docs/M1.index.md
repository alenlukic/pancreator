# M1 index — compact route into MVP and bootstrap scope

This file is the default route for M1 work. Read this before opening full
`docs/PRD.md` or `docs/BOOTSTRAP.md`.

## Default retrieval order

1. `AGENTS.md` for the live operating contract.
2. `src/memory/active/current.md` for current operator pointers, unless simple task
   mode applies.
3. This file for M1 scope and bootstrap routing.
4. `docs/PRD.summary.md` and `docs/PRD.index.md` for product orientation and section
   selection.
5. Full `docs/BOOTSTRAP.md` or `docs/PRD.md` only when the task requires phase-exit detail,
   product-spec depth, or line-anchored citation repair.


## Current bootstrap tracking state

`tesseract.yaml` is the live state tracker. It currently records
`bootstrap.phase: "4"`, `status: phase-4-ratified`, and
`project_root: "."`. The `.` value means this harness is embedded in the same
repository it operates on.

Phase 4 is the right tracking phase because Phase 3 substrate package
implementations and the five MVP pipeline definition files exist, and the US-1
dogfood proof bundle is ratified. Pause/resume/abort evidence is captured, while
Phoenix external trace verification remains a deferred engineering item for
`@tesseract/run-logger` and `tesseract-engineer`. `tess run feature-delivery <inbox-entry>`
now creates the active-work `state.json`, `handoff.md`, `next-prompt.md`, and
`run.log.jsonl` scaffold for manual persona delegation. Stage progress is
recorded only through `tess advance` or explicit `tess repair-state`;
`refresh-prompt` can regenerate `handoff.md`/`next-prompt.md` from the current
ledger. Complete runs emit a bounded librarian handoff for agent-executed
artifact closure via `tess close-artifacts <task-id>`. Do not infer M1 completion from the phase number;
use `docs/BOOTSTRAP.md` Phase 4 and Phase 5 exit criteria for ratification.

conversation mode, greenfield/adopt install paths, pause/resume/abort, package
boundaries, one delivery report artifact, feature indexing, and one library-mode
example.

## Token-economy rule

Agents SHALL NOT open full `docs/PRD.md` or `docs/BOOTSTRAP.md` for routine M1 work if
this file plus `docs/PRD.summary.md` and `docs/PRD.index.md` answer the routing question.
Agents SHALL open exact full-source ranges only when they need authoritative
wording, changed scope, or dual-anchor citations.

When Phase 4 work crosses from planning to execution, agents SHOULD emit a
compact `src/work/<day>/<task-id>/handoff.md` and delegate to the next persona
instead of carrying planner context into implementation.
