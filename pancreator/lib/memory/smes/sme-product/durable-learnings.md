# SME product durable learnings

## 2026-06-12 Command Center post-ship UX feedback

- Prioritize factual trust in control-surface state before shell polish or broad craft cleanup.
- Rank Command Center follow-up work in this order when scope is constrained: truthful reconciled state, fast evidence access, working operator actions, then navigation and visual cleanup.
- Preserve the tested `client/src/services/` layer and existing design-system/component canon while planning UI-facing remediation unless the operator explicitly asks for deeper platform change.
- Treat the Command Center home surface as a 30-second situational-awareness instrument whose job is to answer what needs me, what is running, what failed, and what shipped recently.
