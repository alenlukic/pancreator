# Delivery Report — cursor-token-economy

## Summary
This Feature reduces default Cursor cache-read volume while preserving explicit access to canonical repo surfaces. It adds a root `.cursorindexingignore`, narrows selected Cursor rule globs, introduces `src/memory/handbook/context-economy.md` with a routing entry in `src/memory/handbook/index.md`, splits PRD orientation into `PRD.summary.md` and `PRD.index.md`, and adds `src/internal/tools/context-budget-report.mjs` with matching npm scripts. The implementation keeps `PRD.md`, `BOOTSTRAP.md`, `src/memory/**`, and `src/work/**` reachable for explicit reads and records the follow-through in governed work artifacts. {kind: lines, path: src/internal/work_archive/172997_05-09-26/50909_1000_cursor-token-economy/plan.md, range: [26, 49], contentHash: TBD-on-commit}; {kind: lines, path: src/internal/work_archive/172997_05-09-26/50909_1000_cursor-token-economy/adr-draft.md, range: [44, 76], contentHash: TBD-on-commit}
